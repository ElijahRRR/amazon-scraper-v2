## 结论

| 指标 | 代理 1 (`oegbsfta...`) | 代理 2 (`onka1jdd...`) |
|------|----------------------|----------------------|
| 441 超频上限 | **未触发**（150 并发安全） | **~30 并发触发** |
| 440 带宽上限 | **未触发**（27Mbps 安全） | **未触发**（10Mbps 安全） |
| 最大实测带宽 | **27.1Mbps** | **10.1Mbps** |
| 最大实测 QPS | **24.1** | **4.8** |
| 套餐等级推测 | **高级套餐**（无连接数限制） | **基础套餐**（~20-30 连接限制） |

### 建议

- **代理 1** 适合高并发场景，可以安全地将 `MAX_CONCURRENCY` 设到 40+
- **代理 2** 需要控制在 20 并发以内，当前 config 中 `MAX_CONCURRENCY=16` 设置合理
- 两个代理均**未触发 440 带宽超限**，说明快代理对带宽的计费是按月度/流量总量而非瞬时速率
